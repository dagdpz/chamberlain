# Grid Layout Reference

## Rules

- If ANY parameter has `Layout`, ALL must have `Layout`
- If parameters have `Layout`, the interface MUST include `audioPluginGridLayout`
- `DisplayNameLocation="above"` consumes the row above — allocate in `RowHeight`
- When using `DisplayNameLocation` on any parameter, set it on ALL parameters — each needs its own label row + widget row pair
- `Layout=[row col]` for single cell, `Layout=[r0 c0; r1 c1]` for spanning
- `Padding` format: `[left, bottom, right, top]`
- `vrocker`/`vtoggle` requires exactly 2 enum values; use `'dropdown'` for 3+

## Grid Syntax

```matlab
audioPluginGridLayout( ...
    RowHeight=[20 100 20 60], ...
    ColumnWidth=[100 100 100], ...
    Padding=[10 10 10 30])
```

## Multi-Cell Spanning

Vertical sliders and other tall controls span multiple rows:

```matlab
audioPluginParameter('Gain', ...
    Style='vslider', ...
    Layout=[2,1; 4,1], ...          % spans rows 2-4, column 1
    DisplayName='Gain', ...
    DisplayNameLocation='above')    % label at row 1
```

## Filmstrip Controls

Replace any default control graphic with a custom filmstrip image. Frames are arranged end-to-end (vertical or horizontal) and mapped to the parameter range.

```matlab
audioPluginParameter('Enable', ...
    Style='vtoggle', ...
    Layout=[5,4], ...
    DisplayNameLocation='none', ...
    Filmstrip='toggle_filmstrip.png', ...
    FilmstripFrameSize=[80,60])
```

**Critical rule:** `FilmstripFrameSize` must match the cell dimensions exactly (ColumnWidth × RowHeight of the occupied cell). Mismatched sizes cause clipping or misalignment in the deployed DAW plugin.

For a 2-state toggle: filmstrip is 2 frames tall → total image size = [width, height×2].
For a knob with N frames: filmstrip is N frames tall → total image size = [width, height×N]. Typical knob filmstrips use 63 or 127 frames.

## Background Customization

```matlab
audioPluginInterface( ...
    ..., ...                           % parameters
    audioPluginGridLayout(...), ...    % grid
    BackgroundColor=[0.15 0.15 0.18], ...
    BackgroundImage='panel_bg.png')
```

- `BackgroundColor`: RGB triplet [0–1] or color name. Applied to all space not covered by controls or image.
- `BackgroundImage`: PNG/GIF/JPG. Aligned to grid top-left. Clipped to grid size (not resized).

## Size Calculation

Total UI width = sum(ColumnWidth) + left padding + right padding + (numCols−1) × ColumnSpacing

Total UI height = sum(RowHeight) + top padding + bottom padding + (numRows−1) × RowSpacing

Default spacing is 10 px. Override with `RowSpacing` and `ColumnSpacing` in `audioPluginGridLayout`.

Background images must be at least this size to fully cover the grid.

## Example

Each parameter with `DisplayNameLocation='above'` needs two rows: a label row (~20px) above the widget row.

```matlab
PluginInterface = audioPluginInterface( ...
    audioPluginParameter('Gain', ...
        DisplayName='Gain', ...
        DisplayNameLocation='above', ...
        Mapping={'lin', 0, 1}, ...
        Layout=[2 1]), ...
    audioPluginParameter('Mix', ...
        DisplayName='Mix', ...
        DisplayNameLocation='above', ...
        Mapping={'lin', 0, 1}, ...
        Layout=[2 2]), ...
    audioPluginParameter('Mode', ...
        DisplayName='Mode', ...
        DisplayNameLocation='above', ...
        Mapping={'enum', 'Echo', 'Flanger', 'Chorus'}, ...
        Style='dropdown', ...
        Layout=[4 1]), ...
    audioPluginParameter('Bypass', ...
        DisplayName='Bypass', ...
        DisplayNameLocation='above', ...
        Mapping={'enum', 'Off', 'On'}, ...
        Style='vrocker', ...
        Layout=[6 1]), ...
    audioPluginGridLayout( ...
        RowHeight=[20 100 20 30 20 30], ...
        ColumnWidth=[100 100 100]), ...
    InputChannels=2, OutputChannels=2)
```

Row allocation: rows 1-2 = knob labels + knobs, rows 3-4 = dropdown label + dropdown, rows 5-6 = vrocker label + vrocker.
